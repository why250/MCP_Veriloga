Getting Started with Verilog-A in the Advanced Design System

Advanced Design System (ADS) supports the Accellera 2.3 Verilog-A standard for analog modeling via the latest Verilog-A simulation component set. The Verilog-A devices provide all of the capabilities as built-in components, with the added benefit that you can choose to modify the underlying equations. A number of new devices and models are supplied with ADS to provide both new model capability as well as to provide Verilog-A versions of models that already exist as built-in models.

This documentation provides an overview of the steps necessary to use Verilog-A devices.

Creating a Component for Verilog-A Compact Models

The following section describes the procedure to create a component for Verilog-A compact model file. The symbol can be used in testbench schematic. Sample Verilog-A files are available in $HPEESOF_DIR/veriloga directory.

Before creating a symbol, it is necessary to create a schematic view with the required number of pins.
  1. Create a symbol for the Verilog-A component.
    For more information, see Symbol .
    1. In the Symbol window, click File > Design Parameters .
    2. Set the General Cell Definitions.
      • Specify a description for the component.
      • Specify the component instance name.
      • Uncheck Subcircuit .
      • Select Built-in Component from the Model drop.
      • Specify the Simulate As name.  Use the module name specified in the Verilog-A file.
    3. If you want specific Cell Parameters to be displayed on the schematic, specify them in the Cell Parameters tab.
      Note that if parameters have default values defined in the Cell Parameters tab, these values will override the default values defined in the Verilog-A module.
    4. Click OK .
    5. Save the Symbol.
  2. Create or Open a schematic. Instantiate the component symbol created in step 1 in the schematic.

An example of using VBIC Verilog-A compact model is described in the VBIC_wrk workspace in the Examples/Behavioral_Models/Verilog-A directory. For more information, see Verilog-A Implementations .

Using a Verilog-A Device in a Simulation

To illustrate how Verilog-A components are used and can be modified, a popular GaAs FET model is used in a tutorial workspace to show how a model can be simulated and modified. You can copy the Verilog-A Tutorial example workspace to your home directory or another preferred location using the Operating System.
To simulate a model:

  1. Select File > Open > Example to open the Example directory from the ADS Main window.
  2. Browse to locate the Tutorial_wrk example in the Examples/Behavioral_Models/Verilog-A directory.
  3. Select the example and click Open .
  4. Specify the directory in which you want to install the example and click Choose .
  5. After unarchiving the example, a message box is displayed. Select Yes to open the example.
    After the workspace opens, a ReadMe schematic window will appear as shown in the following figure.
    The Verilog-A Tutorial_wrk ReadMe
  6. The tutorial contains six different examples to illustrate Verilog-A components. Click the PSFETV example button on the left and then push down into the design.
  7. The example consists of a simple DC_FET curve tracer to sweep the device.

  8. Run the simulation.

Modifying a Verilog-A Device

One powerful feature of Verilog-A is that a user can make modifications to the equations that describe the behavior of the device. These changes can be available in the simulator automatically, with no loss of analysis functionality. Many models will be distributed with their source code with the expectation that users will modify the equations for any number of reasons. For example, the user may want the equations to better reflect some aspect of their device behavior, or they may want to delete code that is not necessary to describe their device behavior, thereby improving simulation performance.

In this example, the PSFETV model will be modified slightly. During the simulation, the program searches for the source code based on predefined search paths (discussed in detail later), in the workspace directory, veriloga or as specifically defined using a VerilogA_Load component.

  1. If necessary, copy the workspace Tutorial_wrk from the Examples/Verilog-A directory to a local directory. This tutorial contains a directory called veriloga that includes a file called psfetv.va .
  2. Open the design file tutorial_PSFETV and run a simulation to verify that the results are the same as the previous example.
  3. Using any text editor, open the psfetv.va file.
  4. Find the equation that describes the drain current,
    Id: Id = AREA * Beta_T * (1.0 + LAMBDA * Vdst) * (pow(Vgt, Q) - pow(Vgt - Vdt, Q));
    Purely for demonstration purposes, change the power law relation from "q" to "q/2" in the first power term (this has no physical meaning):
    Id = AREA * Beta_T * (1.0 + LAMBDA * Vdst) * (pow(Vgt, Q/2 ) - pow(Vgt - Vdt, Q));

  5. Save the file and start a simulation.
    The Data Display window now shows very different results (see the following figure) compared to the figure PSFET DCIV Results , demonstrating that the modification to the equation did indeed take effect.
    PSFETV_DCIV Results using Power Law Relation of q/2
